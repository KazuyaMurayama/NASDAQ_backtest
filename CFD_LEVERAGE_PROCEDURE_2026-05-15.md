## 第1章：そもそも「レバレッジ」って何？

### 1-1. 一言で言うと「手持ちのお金より大きな取引ができる仕組み」

**例：10倍レバレッジ**

```
手持ち資金: 10万円
レバレッジ: 10倍
取引できる額: 10万円 × 10倍 = 100万円分のNASDAQ
```

利益も損失も「100万円分」に対して計算されます。

### 1-2. なぜレバレッジを下げたいのか？

レバレッジが高いほど、少しの値動きで資産が大きく増減します。  
たとえば5倍レバレッジなら、NASDAQが10%下がっても損失は50%で済みます（10倍なら100%＝全損）。

**バックテスト結果（参考）：**

| 戦略 | CAGR | 最大下落幅 |
|------|------|-----------|
| 3倍CFD | +23.20% | -44.8% |
| 5倍CFD | +33.33% | -54.9% |
| 7倍CFD | +41.36% | -65.0% |

高リターンを狙うほど下落時のリスクも増えます。自分のリスク許容度に合わせた倍率選びが重要です。

## 第2章：日本の法律での上限（ここだけは必ず知ってほしい）

### 2-1. 個人投資家向けの制限

日本では**金融商品取引業等に関する内閣府令（第123条）**により、個人向けの店頭CFDに次のルールが定められています。

| 商品種別 | 最大レバレッジ（個人） |
|---------|-----------------|
| 株価指数CFD（NASDAQなど） | **10倍まで** |
| 個別株CFD | 5倍まで |
| FX（外国為替） | 25倍まで |

→ **個人口座では、NASDAQのCFDは10倍が上限です**  
→ 5倍・6倍・7倍はすべてこの範囲内 ✅

### 2-2. 法人口座の場合

法人口座はこの規制の対象外になる場合があります（IG証券等）。ただし本レポートは個人口座を前提とします。

## 第4章：5倍・6倍・7倍を実現する方法

### 4-1. 大前提：「◯倍」を選ぶボタンは存在しない

調査の結果、**どのブローカーも「5倍で取引する」「7倍で取引する」と数値を指定するUI機能を確認できませんでした**。

その代わりに、次の2つの方法で実効レバレッジ（実際の倍率）を調整します。

**方法A：ポジションサイズ（枚数・ロット数）を調整する**  
→ OTC CFD向けの主要な方法

**方法B：余剰証拠金を多く積む**  
→ くりっく株365で低レバレッジ運用したい場合

---

### 4-2. 方法A：ポジションサイズで調整（OTC CFD）

#### 計算の仕組み

```
実効レバレッジ = 取引金額合計 ÷ 口座残高（または使用可能な資金）
```

逆に言うと：

```
目標レバレッジで取引するための建玉金額 = 口座の使いたい資金 × 目標レバレッジ
```

#### 具体例：口座に100万円ある場合

| 目標レバレッジ | 必要な建玉金額 | NASDAQが19,000のとき（1枚=100ドル=約15,000円として） |
|------------|------------|----------------------------------------------|
| 3倍 | 300万円 | 約200枚 |
| 5倍 | 500万円 | 約333枚 |
| 6倍 | 600万円 | 約400枚 |
| 7倍 | 700万円 | 約467枚 |
| 10倍（上限） | 1,000万円 | 約667枚 |

> ※ 枚数はブローカー・通貨・指数水準により変わります。実際の計算は後述の手順で行ってください。

#### 証拠金との関係

最大10倍の場合、証拠金率は最低10%（取引金額の10分の1）です。

```
例：建玉金額500万円（5倍レバレッジ）の場合
  → 最低必要証拠金 = 500万円 × 10% = 50万円
  → 残り50万円（口座残高100万円-50万円）は余剰証拠金として安全バッファになる
```

---

### 4-3. 方法B：余剰証拠金を積む（くりっく株365）

くりっく株365はデフォルトが高レバレッジなので、低レバレッジ運用には余剰証拠金が必要です。

**計算例：NASDAQ-100が19,000ポイントの場合（1枚 = 190,000円）**

| 目標レバレッジ | 必要証拠金（1枚あたり） | 最低証拠金（8,960円）の何倍か |
|------------|-------------------|--------------------------|
| 20倍（デフォルト） | 9,500円 | 1.06倍（ほぼ最低額） |
| 10倍 | 19,000円 | 2.12倍 |
| 7倍 | 27,143円 | 3.03倍 |
| 6倍 | 31,667円 | 3.53倍 |
| 5倍 | 38,000円 | 4.24倍 |
| 3倍 | 63,333円 | 7.07倍 |

> 最低証拠金（8,960円）の約4倍を預託すれば、5倍レバレッジに相当します。

## 第5章：具体的な手順（OTC CFD編）

OTC CFDでの手順を高校生向けにステップ形式で説明します。  
（GMOクリック証券・IG証券で一般的な流れ）

### ステップ1：目標レバレッジを決める

バックテスト結果を参考に、自分のリスク許容度で決めます。

```
例：口座に100万円。5倍で運用したい。
```

### ステップ2：目標の建玉金額を計算する

```
目標建玉金額 = 口座残高 × 目標レバレッジ
= 100万円 × 5倍 = 500万円分
```

### ステップ3：現在のNASDAQ指数と1ロット（1枚）の価値を確認する

ブローカーの取引画面で確認します。

```
例：NASDAQ100が19,000ポイント、1ロット = 指数の100倍（ドル）
1ロットの価値 = 19,000 × 100 = 190万円相当（1ドル150円の場合）
```

> ⚠️ ロットサイズはブローカーによって異なります（0.1ロット・1枚など）。  
> 取引画面の「商品概要」「取引要件」を必ず確認してください。

### ステップ4：必要なロット数（枚数）を計算する

```
必要ロット数 = 目標建玉金額 ÷ 1ロットの価値
= 500万円 ÷ 190万円 ≒ 2.6ロット → 2ロット（端数は切り捨て）
```

### ステップ5：証拠金が足りるか確認する

```
最低必要証拠金 = 実際の建玉金額（2ロット） × 証拠金率
2ロットの建玉金額 = 2 × 190万円 = 380万円
最低必要証拠金（証拠金率10%）= 38万円

口座残高100万円 > 38万円 → ✅ 問題なし
実効レバレッジ = 380万円 ÷ 100万円 = 3.8倍（目標5倍より低い → ロット数を増やすか許容）
```

### ステップ6：発注する

計算した枚数でNASDAQ100 CFDを買い注文します。

```
注文タイプ：成行（すぐに約定させたい場合）または指値（価格を指定）
数量：計算したロット数
方向：買い（NASDAQが上がると利益）
```

### ステップ7：建玉完了を確認する ← ここでポジション即時反映

**OTC CFDは成行注文の場合、約定と同時にポジションが即時反映されます。**

## 第6章：ポジション反映スピード

✅ **確認済みの情報**

| 操作の種類 | 反映タイミング |
|----------|-------------|
| 新規ポジション開設（成行注文） | **即時（秒単位）** |
| 新規ポジション開設（指値注文） | **指定価格に達した時点で即時** |
| ポジションの追加（枚数を増やす） | **追加注文が約定した時点で即時** |
| ポジションの削減（枚数を減らす） | **決済注文が約定した時点で即時** |
| 全ポジションの解消 | **決済注文が約定した時点で即時** |

> **まとめ：ポジションの変更は「約定した瞬間」に即時反映されます（営業日をまたがない）**

⚠️ **要確認の情報**

| 項目 | 状況 |
|------|------|
| 夜間・早朝（市場休止中）の注文の扱い | ブローカーにより異なる。各社の取引時間を要確認 |
| 分割発注時のスリッページ（大口の場合） | 市場流動性により変動。大口注文は価格が不利になる場合あり |

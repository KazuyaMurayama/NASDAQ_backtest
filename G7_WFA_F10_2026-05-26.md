## 目的

F10 ε-deadband スイープ (`F10_EPSILON_DEADBAND_2026-05-26.md`) で発見された 2点:

1. **Trades/yr 計測バグ修正**: F8-R5 元実装の Trades/yr=182 は誤計測。`lev_mod` (LT2 bias 込みで連続変化) を取引イベントとしてカウントしていた。正しい基準は `lev_raw` (`simulate_rebalance_A` の discrete output)。修正後の実 Trades/yr ≈ 52（ユーザー許容範囲）。
2. **ε=0.015 採用候補**: Sharpe=+0.935, CAGR_OOS=+36.84%, Trades/yr=51.6 — deadband なしの F8-R5 と指標がほぼ同等。

G5 WFA で F8-R5 (ε=0) は PASS (CI95_lo=+27.92%, WFE=+1.208) を達成済。G7 は **ε=0.015 適用後も同等の安定性が保たれるか** を正式確認する。

```python
# F10 と同一 (src/f10_epsilon_deadband.py 参照)
cur_tilt = 0.0
for i in range(n):
    raw_tilt = clip(TILT * (raw_a2[i] - 0.15) * (1 - raw_a2[i]), 0, cap_eff[i])
    if not bull_mask[i]: raw_tilt = 0
    if i == 0 or abs(raw_tilt - cur_tilt) >= eps:
        cur_tilt = raw_tilt   # ε 以上の動きのみ反映
    confirmed[i] = cur_tilt

wn_tilted = wn_A + confirmed
wb_tilted = clip(wb_A - confirmed, 0, wb_A)
```

**重要な注意**: F10 ε=0.015 のバックテスト IS-OOS gap は -4.31pp で、E4 (-1.81pp) より広い。これは IS 期間 (1980-2021) で OOS 期間 (2021-) よりも過剰にフィットしている兆候であり、将来のパフォーマンスがバックテスト水準を下回るリスクを示唆する。

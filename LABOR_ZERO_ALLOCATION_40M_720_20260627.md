# 引退後・労働補填ゼロ設計の網羅探索 — 資産4,000万円・年間支出720万円

作成日: 2026-06-27
最終更新日: 2026-06-27

> **依頼（ユーザー 2026-06-27）**: 資産4,000万円・年間支出720万円。**できる限り労働（外部就労）による補填をしたくない**。現行案＝「scale2.0 に3,000万円運用、2,000万円を割ったら予備1,000万円を補填」がかなり良いと考えているが、もっと良い方法がありそう。**ゴール＝1975〜2005年のどの開始年でも、その後20年間、1度も労働補填せずに済む（＝予備込み4,000万が尽きるまで外部就労なし）方法を網羅的に探す**。
>
> **探索（ユーザー確定）**: 戦略 × 運用額（4,000万のうち何を運用に回すか）× 補填ルール（閾値・補填額・予備の運用）を全変数スイープ。比較戦略は本セッションの主要候補を網羅（P09_STR sc1.0〜2.2 ＋ N4 ＋ X4 ＋ X4相当 sc1.4/1.6/1.8/2.2 ＋ NASDAQ1倍）。
>
> **モデル定義**: 総資産＝運用スリーブ＋予備スリーブ。支出720万円/年は運用から取り崩し。**予備＝現金（無成長）**を、運用残高が閾値を割ったら補填ルールで運用へ移す（＝自己資産のリバランス。労働ではない）。**労働年（ゼロにしたい対象）＝補填後も総資産が720万円の支出を賄えない年（＝予備込み4,000万を使い切り外部就労が必要）**。運用スリーブにのみ市場リターン（税後×0.8273）を適用。
>
> **一次根拠**: `src/audit/labor_zero_allocation_sweep_20260627.py` / `audit_results/labor_zero_allocation_sweep_20260627.csv`（325組合せ全グリッド）/ `audit_results/labor_zero_winners_20260627.csv`。年次リターンは既存検証済CSV（scale dial・x4n4・x4equiv）を使用。

## ⭐ 結論（先出し）：年720万円・資産4,000万円（取り崩し率18%）では「労働補填ゼロ」は達成不能。ただし**惜しい**

**325組合せ（戦略13×運用額×補填ルール）を全探索した結果、1975〜2005年の全31開始年で労働補填ゼロを満たす設計は1つも存在しない。** 最良でも12年（特定の1開始年に集中）の労働補填が残る。原因は明確で、**年720万円÷4,000万円＝18%の取り崩し率が高すぎる**こと。レバレッジ戦略でも、運用開始直後に下落が来ると予備を使い切り、その後どれだけ高リターンが続いても回復不能（シーケンスリスク）。

**ただし「あと一歩」**：労働補填ゼロを満たせる上限は、**支出なら年約670万円（16.8%）／資産なら約4,290万円**。あなたの目標（720万円・4,000万円）との差は **年約50万円 または 資産約290万円**。

### 重要な3点
1. **現行案は最良ではない（むしろ悪い部類）**: 「sc2.0・運用3,000万・予備1,000万・2,000万割れで1,000万補填」は **労働補填33年・4開始年で破綻**。運用額を増やしすぎ＋予備が薄く、補填が後手に回る。
2. **最良設計＝sc2.2・運用2,000万／予備2,000万（50:50）・運用が2,000万を割ったら予備を全額投入**: 労働補填12年・**31開始年中30年は完全に労働補填ゼロ**・失敗は1988年開始のみ・終端資産中央値も最高（約75億円）。
3. **意外な発見＝この目的では「高レバ・高CAGR」が「低MaxDD」に勝つ**: 18%もの高取り崩しでは、**取り崩しを上回る速さで資産を増やすCAGRが最重要**。MaxDDが浅いX4相当やsc1.6より、CAGRの高い**sc2.2が労働補填を最小化**する（取り崩しレースではDD防御より成長力）。

> ⚠ 本探索は過去系列1本（1975-2025）での結果であり将来非保証。合否ラベルは付さず、生の計測値（労働補填年数・破綻数・終端資産）で提示する。

## 2. 現行案 vs 最良案 — 直接比較

| 観点 | 現行案 | 最良案 |
|:---|:---|:---|
| 戦略 | sc2.0 | **sc2.2** |
| 運用/予備 | 運用3,000万/予備1,000万 | **運用2,000万/予備2,000万** |
| 補填ルール | 2,000万割れで1,000万補填 | **2,000万割れで予備全額（2,000万）投入** |
| **労働補填年数** | **33年** | **12年** |
| 労働が出る開始年 | 4開始年（1988/2000/2004/2005） | **1開始年（1988のみ）** |
| 破綻（総資産0） | 4開始年 | **1開始年** |

- **現行案は労働補填33年・4開始年で破綻**＝「かなり良い」とは言えず、むしろ本探索の中位以下。**運用額が多すぎ（3,000万）＋予備が薄い（1,000万）＋補填が分割（1,000万ずつ）**で、初期下落時に運用スリーブを守りきれない。

- **改善の本質**: ①戦略をsc2.0→sc2.2へ（CAGRを上げ取り崩しレースに勝つ）②運用を3,000万→2,000万へ減らし予備を2,000万へ厚く③補填は分割でなく**予備全額を一括**で運用へ戻す（回復局面に最大限乗る）。

## 3. なぜゼロにできないか — 1988年開始の破綻パス（最良案でも破れる唯一の開始年）

最良案（sc2.2・運用2,000万/予備2,000万・2,000万割れで全額投入・支出720万）でも、**1988年開始だけは破綻**する。そのパス：

|--:|--:|--:|--:|--:|:--|
| 1988 | −24.4% | 968 | 2,000 | 2,968 | （初年度に大幅下落） |
| 1989 | +56.5% | 3,516 | 0 | 3,516 | **予備全額投入**（運用が2,000万割れ） |
| 1990 | −29.9% | 1,960 | 0 | 1,960 | |
| 1991 | +68.2% | 2,085 | 0 | 2,085 | |
| 1992 | +65.3% | 2,257 | 0 | 2,257 | |
| 1993 | +3.6% | 1,593 | 0 | 1,593 | |
| 1994 | −9.2% | 793 | 0 | 793 | |
| 1995 | +149.4% | 182 | 0 | 182 | （取り崩しで枯渇寸前） |
| 1996 | +41.1% | 0 | 0 | 0 | **労働開始**（総資産が720万を賄えず） |
| 1997〜2007 | +51.8〜+244.7% | 0 | 0 | 0 | **労働継続**（資産は既にゼロ・高リターンも乗れず） |

**破綻の正体＝レバレッジ×高取り崩しのシーケンスリスク**:
- 1988年は初年度 −24.4%（sc2.2の高レバが下落を増幅）で、**2年目に予備を全額使い切る**。
- その後 1990(−29.9%)・1993(+3.6%)・1994(−9.2%) の凡庸〜下落が続く間も**毎年720万を取り崩す**ため、運用スリーブが1996年にゼロへ。
- **皮肉なことに1995-1999に+149/+41/+111/+204%の爆発的上昇が来るが、資産は既に枯渇していて乗れない**。これがシーケンスリスクの本質＝**「早期の下落＋予備枯渇」は、その後どれだけ高リターンでも回復不能**。
- 18%もの高取り崩しでは、このパターンを完全には避けられない。レバを下げてもCAGRが落ち別の開始年が破れる（だからゼロは全戦略で不能）。

## 4. 「労働補填ゼロ」を達成する条件 — フロンティア（あと一歩）

ゼロが現状不能なら、「**何を変えればゼロになるか**」を定量化する。最良戦略（sc2.2・運用50%/予備50%・閾値2,000万・全額投入）で、全31開始年が労働補填ゼロになる境界：

### 4.1 支出を下げる場合（資産4,000万を固定）
| 戦略 | 労働補填ゼロの上限支出 | 取り崩し率 | 目標720万との差 |
|:---|:--:|:--:|:--:|
| **sc2.2** | **年 約671万円** | 16.8% | **−約49万円/年** |
| sc2.0 | 年 約657万円 | 16.4% | −約63万円/年 |
| N4 | 年 約620万円 | 15.5% | −約100万円/年 |
| X4 | 年 約617万円 | 15.4% | −約103万円/年 |

→ **支出を年720万→約670万（月60万→月56万）に約7%下げれば、sc2.2で全31開始年が労働補填ゼロ**になる。

### 4.2 資産を増やす場合（支出720万を固定）
| 戦略 | 労働補填ゼロの最小総資産 | 取り崩し率 | 目標4,000万との差 |
|:---|:--:|:--:|:--:|
| **sc2.2** | **約 4,290万円** | 16.8% | **+約290万円** |
| sc2.0 | 約 4,380万円 | 16.4% | +約380万円 |
| N4/X4 | 約 4,650万円 | 15.5% | +約650万円 |

→ **資産を4,000万→約4,290万（+290万）にすれば、sc2.2で全31開始年が労働補填ゼロ**になる。

### 4.3 含意
- **両フロンティアは「取り崩し率 約16.8% が sc2.2 の労働補填ゼロ天井」**で一致（支出671万/4,000万＝16.8%、720万/4,290万＝16.8%）。あなたの18%はこの天井をわずかに超えている。
- **最小コストの是正は「支出を年49万下げる」または「資産を290万増やす」**。どちらか一方で達成。
- レバをこれ以上上げてもゼロにはできない（sc2.4等はMaxDDが深く別の開始年が破れ、天井は改善しない）。**ゼロの鍵は取り崩し率を16.8%以下に収めること**であり、戦略選択（sc2.2）と配分（50:50・全額補填）は既に最適化済み。

## 5. 推奨アクション

労働補填を**実質ゼロに近づける**現実的な選択（手間最小・あなたの優先順位）：

1. **設計を現行案から最良案へ変更**（無コストで労働補填33年→12年・破綻4→1開始年）:
   - 戦略 **sc2.0 → sc2.2**、運用 **3,000万 → 2,000万**、予備 **1,000万 → 2,000万**、補填 **2,000万割れで予備全額（2,000万）一括投入**。
   - これだけで31開始年中**30年は完全に労働補填ゼロ**。残るリスクは1988型（初年度大幅安＋以後低迷）の1開始年のみ。
2. **完全ゼロを求めるなら、以下のどれか一つ**（取り崩し率を16.8%以下へ）:
   - **支出を年720万→約670万（月約56万）に下げる**（−7%）、または
   - **運用開始時資産を約4,290万へ（+290万）**、または
   - **支出を一時的に絞れる柔軟性を持つ**（1988型の初期下落局面の数年だけ支出を660万へ落とせれば、外部就労なしで凌げる＝「労働」ではなく「節約」での補填）。
3. **手間（予備補填イベント）も最小化したいなら**: 最良案（50:50・全額補填）は予備補填が平均的に少なく（多くの開始年で0〜1回）、補填の判断も「2,000万割れで全額」の1ルールのみ＝運用の手間も小さい。

> **最重要メッセージ**: 「scale2.0・運用3,000万・2,000万割れで補填」という現行案は、実は労働補填が多い設計（33年・4破綻）。**運用を減らし予備を厚くし（50:50）、戦略をsc2.2へ上げ、補填を全額一括にする**だけで劇的に改善する。ただし**年720万÷4,000万＝18%取り崩しは、過去最悪の開始年（1988）を完全には救えない**。完全ゼロには支出−49万/年 か 資産+290万 のどちらかが必要。

## 7. 脚注・前提・限界
- **労働補填の定義**: 補填後も総資産（運用+予備）が当年支出720万を賄えない年＝外部就労が必要な年。予備の運用への補填（リバランス）は労働ではない。
- **予備＝現金（無成長）**: 予備スリーブは市場運用せず名目固定（保守側）。予備も1倍投信等で運用すれば結果は改善しうるが、本探索では現金前提（ユーザー現行案に忠実）。
- **取り崩し順序**: 年初に補填判定→支出（運用優先・不足分は予備）→運用スリーブに市場リターン適用。閾値ちょうどは補填しない（厳密未満）。
- **名目固定・通貨中立**: 支出720万・資産4,000万・閾値はすべて名目（インフレ非連動）。リターンはUSD名目・為替中立・税後×0.8273。

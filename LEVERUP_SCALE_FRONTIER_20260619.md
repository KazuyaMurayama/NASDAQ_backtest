- **頑強性列で 1.80/2.00 は ❌過学習疑い**: IS-OOS gap が **6.16/7.57pp と 5pp超**（WFE/CI95/CPCVは良好だが、高レバほど IS優位が拡大＝過学習/レジーム signalが強まる）。1.40/1.60 は ⚠条件付（gap 3-5pp）。**＝レバを上げるほど"IS期間に都合よく効いていた"傾向が強まる、という警告**。
- **Sharpe_Full はレバ上げで一貫低下**（1.07→1.06→1.04→1.03、全て◎だが★未満）＝**純レバ（risk-adjustedは改善せず、むしろ僅かに悪化）**。**P09_C1 は Sharpe_Full +1.13（★）で6系列中リスク効率トップ**。
- **最悪単日テールは比例超で悪化**（−19.5%→−28.3%、全2016 Brexit日）。Worst5Y は 1.60以上で全てマイナス（−0.31〜−1.93%）。
- ⚠ **対V7/対B3a bootstrap は全水準で95%非有意**（点推定は上だが「有意に上回る」とは言えない・家系共通）。

## 4. 脚注・前提
- **コスト**: TQQQ borrow=SOFR+0.50%・TER、>3x超過分 k365 EXCESS_EXTRA=0.25%/yr、売買コスト（Trades/yr連動）、C1 OUTフィル（Gold常時＋Bond@mom252>0・逆ボラ63日・TER込み）。全て日次でNAVに反映済。
- **税後（×0.8273）**: 譲渡益税の effective乗数（§1⓽と同基準）。§1は CAGR水準・§2/§3 は**年次リターン水準**に適用（タイミングが僅かに異なるため §1のOOS/IS CAGR と §2.1の全期間複利は厳密一致しない）。年次税後は損失年も同率（損益通算/繰越を簡略化、保守側）。
- ⚠ **NASDAQ 1倍B&H は売却時まで課税繰延**＝年次×0.8273は過剰課税。**実際のB&H手取りは年次税後より高く税前に近い**（終端で総利得に1回課税）。高回転戦略（Trades 29-35/年）は毎年利益確定が現実的で年次×0.8273が妥当。
- ⚠ **§1 頑強性セルの IS-OOS gap は税前CAGR差（pp）**（gap自体に税変換は不要なため）。判定（gap>3pp で⚠／>5pp で❌）は税前基準で行う。
- **本版は思考実験（DD不問）**: scale1.60/1.80/2.00 は標準ハードベト（MaxDD<−50%、scale2.00はRegime_min<−10%も）に抵触する**参考値**で採用候補でない。標準内（全ベトPASS）は scale1.40・本命候補は別途（scale1.50＝前版／P09_C1）。
- **P09_C1 の実効L>3 日 ≈ 6.1%**（正典 [LEVERUP_SWEEP §6.1](LEVERUP_SWEEP_RESULTS_20260612.md) 訂正値）だが P09 のコストモデルは超過分に k365 課金なし（excess_days=0）。
- **P09_C1 頑強性入力（§6.1/REGISTRY確定値）**: WFE=0.989・CI95_lo=+18.96%・CPCV=+14.15%・Regime_min=−0.08%・IS_OOS_gap=+2.56pp・対V7 P_min=0.820。
- 一次データ: `src/audit/leverext_high_stage1_20260618.py`（scale1.40/1.60）、`leverext_high_sc180_200_20260620.py` / `audit_results/leverext_high_sc180_200_20260620.csv`（scale1.80/2.00）、`leverup_b1c1_20260612.py`（P09_C1）、`annual_returns_sc180_200_20260620.csv`（年次・税前ソース）。年次税後・統計は実NAV税前×0.8273で機械生成（Python集計）。Stage-1 詳細は [LEVERUP_EXTENSION_RESULTS_20260616.md](LEVERUP_EXTENSION_RESULTS_20260616.md)。

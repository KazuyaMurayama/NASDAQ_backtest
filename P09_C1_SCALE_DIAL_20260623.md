> **このレポート（旧 default マップ版）は誤った下位戦略を採用していたため廃止されました。**
>
> 旧版は P09_C1 の**素の default マップ** `{Q0:1.20, Q1:1.10, Q2:1.00, Q3:1.00}` を拡張したダイヤルでしたが、これは **CURRENT_BEST_STRATEGY.md L72/L270 が採用検討中の正典系列（strong マップ）ではなく、CAGR/MaxDD効率で劣る下位戦略**でした（例: scale2.0 で OOS +23.94% / MaxDD −58.58% ＝ strong の +29.11% / −61.63% に劣後）。
>
> **正しい現行版 → [P09_STR_SCALE_DIAL_20260623.md](P09_STR_SCALE_DIAL_20260623.md)**（strong マップ `{1.60,1.50,1.10,1.00}`・`B1_SCALE_FRONTIER_20260621.md` の `P09_STR_S*` と同一基準・FRONTIER値を4桁再現）。
>
> **差し替えの経緯**: ユーザー指摘により、本 default 版が `B1_SCALE_FRONTIER_20260621.md` の `P09_STR_S2.0`(+35.38%/+29.11%) と大きく乖離していた原因を調査。乖離は「同じ "P09×scale" という名前で **boost マップが別物**（default vs strong）」であることが両生成コード・CSVの `v7_map` 列から判明。**名前に対する忠実さでは default 版が "素の P09_C1"** だったが、**採用検討中の正典系列・CAGR/MaxDD効率では strong 版が優位**のため、strong 版へ差し替えた。
>
> **命名の是正（混同防止）**: strong マップ系列 = **P09_STR**、default マップ系列 = **P09_DEF** と別称し、両者を "P09×scale" で一括りにしない。
>

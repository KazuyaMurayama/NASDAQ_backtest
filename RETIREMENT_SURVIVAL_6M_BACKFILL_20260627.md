> - **(a)** 運用資産残高が **2000万円を下回った**年は、その年の支出600万円を**ゼロ**にする。
> - **(b)** 考え方: 相場悪化時に**外部で600万円を稼ぎ**、支出分を相殺する（運用資産を取り崩さない）。
> - **(c)** 年始時点で依然2000万円未満なら、その年も支出ゼロ＝**2000万未満である限り毎年、何年でも支出ゼロを継続**。

## ⭐ 結論サマリ：X4/N4 は sc1.8〜2.0 と同等の「ほぼ補填不要」ゾーンに入る

補填ルール（残高2000万未満の年は外部で600万稼ぐ）を入れると、**全8系列・全31開始年で破綻がゼロ（31/31生存）**。NASDAQ 1倍ですら破綻しない。問われるのは「破綻するか」ではなく**「何年、外部で働いて支出を相殺する必要があったか（補填年数）」**。

- **X4 / N4 の補填年数は合計10年・平均0.32年・最大3年**＝**sc1.8（10年）・sc2.0（9年）とほぼ同じ**。31開始年のうち**24開始年で補填ゼロ**＝予定どおり600万を取り崩しても2000万ラインをほぼ割らない。
- これは X4/N4 が **IN脚に低相関の1倍Gold/Bondを混ぜてMaxDDを下げつつCAGRをsc2.0近傍に保つ**（[レバ版リスクパリティ・コア](SC2_DD_REDUCTION_RESULTS_20260626.md)）設計の帰結。早期のバッファ蓄積はsc1.8〜2.0級で、補填負担も同水準に収まる。
- **20年後資産（生存分・中央値）は X4 約38.0億・N4 約35.4億**＝**sc1.8（30.3億）と sc2.0（49.1億）の中間**。CAGRがsc2.0を僅かに下回るぶん終端資産も間に位置する。
- 補填負担だけ見れば **scale1.6+（sc1.6/1.8/2.0/N4/X4）はどれも合計9〜11年・最大2〜3年**で実質横並び。**生存も補填もこのゾーン内では差は小さく、選択は終端資産（レバ水準）とMaxDD/下落リスクのトレードオフで決まる**。

> ⚠️ 本レポートは「引退後の取り崩し耐性（破綻・補填年数）」の比較であり、合否ラベルは付さない（生の計測値のみ）。X4/N4 の標準10指標・MaxDD・頑健性の評価は [X4_N4_VERIFICATION_20260626.md](X4_N4_VERIFICATION_20260626.md)（独立2層検証）を参照。現行の分析上ベスト戦略は P09_STR_sc1.6（[CURRENT_BEST_STRATEGY.md](CURRENT_BEST_STRATEGY.md)）であり、本シミュレーションは分析であって本番稼働ではない。

## 4. X4/N4 を入れて何が変わったか（前版 [scale2.2/2.4] 版との対比）

| 観点 | 前版（scale2.2 / 2.4） | 本版（X4 / N4） |
|:---|:---|:---|
| 追加レバ系の性格 | **一律レバ増**（strong-map × scale2.2/2.4・IN脚100%NASDAQ） | **IN脚にGold/Bondを混ぜたレバ増**（risk-parity-lite・MaxDD低減狙い） |
| 補填年数（合計） | sc2.2=8年 / sc2.4=8年 | N4=10年 / X4=10年 |
| 補填ゼロ開始年 | sc2.2=25 / sc2.4=25 | N4=24 / X4=24 |
| 20年後資産中央値 | sc2.2≈72.7億 / sc2.4≈103.3億 | N4≈35.4億 / X4≈38.0億 |
| 設計意図 | レバを上げ続けて終端資産を最大化 | sc2.0級CAGRを保ちつつ**MaxDDと下落リスクを下げる** |

- **scale2.2/2.4 は「とにかくレバを上げて終端資産を最大化」**する方向で、補填は最少（8年）だが MaxDD は sc2.0 よりさらに深い（[scaleダイヤル](SCALE16_PROMOTION_TO_BEST_20260626.md)）。
- **X4/N4 は逆向き**＝sc2.0近傍のCAGRを保ったまま MaxDD/下落リスクを下げる候補。補填負担は scale2.2/2.4 より僅かに多い（8→10年）が、それは「同じレバ水準の一律版より終端がやや低い＝バッファ蓄積がやや遅い」ぶん。**取り崩し耐性（補填）と終端資産では scale2.2/2.4 が上だが、MaxDD/シャープ/下落リスクでは X4/N4 が上**というトレードオフ。本版は後者を比較対象に入れ替えた。

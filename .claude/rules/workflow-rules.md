# ワークフロー・エージェントルール

## タスク管理
- `tasks.md` を作成・維持し、タスクを一元管理する。
- 新セッション開始時に `tasks.md` を参照し、未完了タスク・優先度・進捗を即座に把握する。
- タスクは細かい単位に分割し、完了のたびにコミットしながら進める（途中停止リスクを最小化）。

## モデル使い分け
| 用途 | モデル |
|------|--------|
| 計画・分析・高度な論理推論 | **Opus** |
| 実行・サブタスク・その他大部分のステップ | **Sonnet** |

## AIエージェントによる品質チェック
- アウトプットに対して自己品質チェックを行い改善する。
- タスクの性質・目的に合わせて効果的なエージェント構成とラウンド数を計画してから実行。
- エージェント例（必要に応じて選択）:
  批判思考 / 論理思考 / 水平思考 / AI専門家 / 戦略家 / アーキテクト /
  エンジニア / デザイナー / PM / 営業 / マーケティング / ユーザー / コンサルタント
- ⚠️ **一度に起動するエージェントは最大4体まで**

## タイムアウト対策
- タスクを細かいサブタスクに分割し、各ステップで保存・コミットしながら進める。
- 長時間処理が見込まれる場合はチェックポイントを明示的に計画してから実行。
- タイムアウトが起こりうる原因を予測し、対策の妥当性もチェックしてから実行。

## ファイルインデックス管理
- `FILE_INDEX.md` を作成・維持し、毎回プロンプト受信時に参照する。
- 品質チェック項目:
  1. **網羅性**: main はもちろん、ブランチのすべてのファイルを記述する。
  2. **優先度の明示**: 同種ファイルが複数ある場合、更新が新しいものを優先確認とする。
     優先度が不明確な場合はファイル名に最終更新日（`YYYY-MM-DD_xxx.md`）を含める。

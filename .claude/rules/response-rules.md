# 回答ルール

## 回答フォーマット
1. **回答冒頭にプロンプト要約を必ず明記**してから本題に入る。
2. 論理的・客観的・事実ベースで回答する。おべっか・楽観コメントは禁止。
3. 意外な点・反直感的な情報は強調する（太字・⚠️等を活用）。
4. ユーザーに判断を求めるときは選択肢と推奨案をセットで提示。
5. できるだけClaude Code側でタスク実行し、ユーザー側タスクを最小にする。
   - ユーザーへのアクション依頼は手順最小・漏れなし・正確に。
6. セッション継続が難しくなったら、その旨＋新セッション用引き継ぎプロンプトを回答に含める。

## 成果物の報告（厳守）
- 完成ファイルは必ずGitHubのハイパーリンクを添えて報告する。
- ワンクリックで開けるリンク形式:
  ```
  https://github.com/KazuyaMurayama/NASDAQ_backtest/blob/<branch>/path/to/file.md
  ```

## 出力フォーマット
- mdファイルを優先。Google スプレッドシート・Word・PPT はユーザー指示があれば使用。
- Word（.docx）・PPT（.pptx）出力時はページ番号を必ず自動付与。

# 労働補填ゼロ v2 のホールドアウト検証【再現性の懸念を検証】— 2006-2020開始で破綻する

作成日: 2026-06-28
最終更新日: 2026-06-28

> **検証の動機（ユーザー 2026-06-28）**: 労働補填ゼロ設計を **v2** と命名・固定。最大の懸念は「インフレ」ではなく **再現性**＝「v2 は 1975-2005 開始（30年データ）に過剰適合しているのではないか」。**v2 探索に使っていない 2006-2020 開始**で、データのある **2025 年まで・各開始年で 5〜20 年**走らせ、(a) 生き延びるか (b) トータル資産の最小到達点はいくらか、を検証する。
>
> **⭐ 結論（先出し）**: **懸念は的中した。v2 は 2006-2020 開始のうち 3 開始年（2012・2013・2014）で破綻する**（運用＋予備が完全にゼロになり、7〜9 年の労働補填が発生）。残り 12 開始年は労働ゼロで生存。**＝v2 は無条件に再現する設計ではなく、「最悪の暴落が運用開始の直後（1〜2 年目）に来る」という 1975-2005 に固有のパターンに依存している**。2012-2014 開始は暴落（2015 年 sc2.6 −38.2%）が**予備を撃ち尽くした後**に来るため、二の矢がなく枯れる。
>
> **v2 はベスト版ではない。v3/v4 で「最悪の暴落が後から来る系列」への頑健性を作り込む必要がある**（具体策は §4）。

## 3. なぜ 2012-2014 開始だけ破綻するのか — 過剰適合の正体

### 3.1 v2 の暗黙の前提＝「最悪の暴落は運用開始の直後に来る」
v2 は **予備を「1回だけ全額投入」**する（一方向・1 発の弾）。この設計が IS（1975-2005）で機能したのは、**どの開始年でも最大の運用ドローダウンが運用初期（1〜2 年目）に来た**ため——初期に 1 発撃てば、その後の回復ラリーに乗れた。

**しかしこれは「最悪の暴落が最初に来る」という IS 固有のパス形状に依存していた**。2012-2014 開始では、**最大の暴落（2015 年 sc2.6 −38.2%）が運用 2〜3 年目＝弾を撃ち尽くした後**に来る。

### 3.2 2013 開始の実トレース（最悪ケース）
```
2013: sc2.6 +29.3%  bond −9.6%               → 運用 8.79M / 予備 23.49M（運用が14Mを割る）
2014: sc2.6  +2.4%  bond +19.0% [予備を全額投入] → 運用 25.69M / 予備 0     ← 弾を撃ち尽くす
2015: sc2.6 −38.2%  bond −2.3%               → 運用 11.42M（暴落・債券も助けず）
2016: sc2.6  −2.0%  bond +0.6%               → 運用  4.14M
2017: 運用 4.14M で 720万を賄えず            → 労働（以後ゼロ・大相場に乗れない）
      2017 +81% / 2020 +213% / 2023 +70% は手遅れ
```

**3 つの不運が重なった**:
1. **補填（予備の全額投入）が暴落の前年（2014）に起きてしまう**＝弾を早撃ちして撃ち尽くす。
2. **その直後の 2015 が −38.2% の大暴落**＝補填で太らせた運用がまた急減。
3. **2015 は債券も −2.3%**＝v2 の生命線「株安で増える債券予備」が、もう予備が空なので効かない（＋そもそも 2015 は債券も弱い）。

### 3.3 IS にはなくホールドアウトにある「v2 キラー」パターン
- **sc2.6 と債券が同時にマイナスの年**: 1977・1994・**2015**・2018。このうち **2015 は大暴落（−38.2%）かつ債券もマイナス**で、補填後に予備が空なら二の矢がない。
- IS の暴落（1990・2000・2008 等）は**債券が強くプラス**（2008 +26.3%）だったため予備が機能した。**2015 型の「株も債券も弱い・しかも開始数年後」は IS にほぼ存在しなかった**。
- ＝v2 は「暴落時に債券が必ず助ける」「暴落は最初に来る」という、**たまたま IS で成立した 2 つの仮定**に乗っている。

## 4. 含意 — v2 はベストでない・v3/v4 の設計要件

**v2 の「労働ゼロ」は 1975-2005 開始に固有であり、再現性は限定的**（ユーザーの懸念が正しい）。本番運用は「いつ引退するか＝どの開始年か」を選べないため、**特定の開始年で 7〜9 年の労働補填**が出る設計は採用できない。

### v3/v4 で作り込むべき頑健性（仮説・要検証）
1. **予備を「1回全額」でなく「分割投入」する**: 1,400万割れで半分、さらに割れたら残りを投入。**弾を温存し、暴落が後から来ても二の矢を残す**（2012-2014 破綻の直接の対策）。
2. **補填の閾値・タイミングを暴落の深さに連動させる**: 一律「14M割れ」でなく、急落時ほど厚く投入。
3. **予備資産の多様化**: 債券だけだと 2015 型（株債同時安）に弱い。金・現金・SOFR の混合で「債券も弱い暴落」に備える。
4. **運用スリーブのレバを下げる選択肢**: sc2.6 の −38% 級ボラが破綻の引き金。sc2.4/2.2 で初期ドローダウンを浅くし、その分予備を厚く/分割で持つトレードオフを再探索。
5. **評価を全開始年（1975-2020）×可変ホライズンで行う**: v2 は IS 31 開始年でしか評価していなかった。**v3 以降は 2006-2020 を必ず評価に含める**（本レポートの検証を恒久化）。

> **方法論教訓**: サバイバル sim の「全開始年で労働ゼロ」は、**評価した開始年集合に強く依存**する。labor-zero を「達成」と呼ぶ前に、**探索に使っていない開始年（ホールドアウト）で必ず再検証**せよ。v2 はこの検証を欠いていた。

> **🔜 v3 で決着（2026-06-28）**: 上記要件で多ラウンド探索した結果が [LABOR_ZERO_V3_ROBUST_20260628.md](LABOR_ZERO_V3_ROBUST_20260628.md)。**固定支出720万のままでは全46開始年 labor-zero は構造的に不能**（静的1,050構成＋完全予知オラクルまでで到達不能・2012-2014は18%取り崩しに対し系列が厳しすぎる）。**ただし支出を資産連動（保証フロア＋好機トップアップ）にすれば全46開始年で労働0・破綻0が達成可能**（最良＝平常720万・最悪期360万）。運用設計（v2）は不変・変えたのは支出ルールのみ。

## 6. 脚注・前提
- **モデル**: 総資産＝運用スリーブ（sc2.6・税後）＋予備スリーブ（1倍債券・税後）。支出 720万を毎年厳密取り崩し（運用優先・不足は予備）。補填＝運用<1,400万で予備を全額投入（自己資産リバランス・労働でない）。労働年＝補填後も総資産が 720万を賄えない年。
- **ホールドアウトの定義**: v2（および Round-1/Round-2 探索）は開始年 1975-2005 のみで評価・調整された。2006-2020 開始は**一度も探索・調整に使われていない真のホールドアウト**。
- **データ制約**: sc2.6・債券とも 1975-2025（51年）。2006-2020 開始の 20 年フルは取れない（2006 開始のみ 20 年・以降は逓減）。短いホライズンほど「まだ破綻が顕在化していないだけ」の可能性に留意（2020 開始は 6 年で生存だが 20 年生存の保証ではない）。
- **後知恵バイアス**: 過去系列 1 本。本検証は「IS への過剰適合」を暴くものだが、ホールドアウト自体も単一史実パスであり、将来保証ではない。
- **本検証は分析であり本番運用助言でない**。現行の分析上ベスト戦略は P09_STR_sc1.6（[CURRENT_BEST_STRATEGY.md](CURRENT_BEST_STRATEGY.md)）。

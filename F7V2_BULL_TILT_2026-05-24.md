| **bull mask** | raw_a2 > THRESHOLD (0.15) |
| **Base config** | E4 採用: k_lo=0.1, k_hi=0.8, vz_thr=0.7, LT2-N750, mode B |
| **IS** | 1974-01-02 〜 2021-05-07 |
| **OOS** | 2021-05-08 〜 |

**メカニズム**:
```
bull_mask    = raw_a2 > THRESHOLD                                   # 0.15
tilt_amount  = tilt × (raw_a2 - THRESHOLD) × (1 - raw_a2)
tilt_amount  = clip(tilt_amount, 0, TILT_CAP)                       # cap = 0.15 / 0.20
wn_tilted    = wn_A + tilt_amount         (bull 日のみ)
wb_tilted    = max(wb_A - tilt_amount, 0) (bond からの振替, 負禁止)
```
F7 (cap=0.10) で頭打ちになっていた高確信 bull 日（raw_a2 ≈ 0.5 付近）の
tilt_amount を、cap 引き上げにより本来のピーク値まで通す。

**仮説の根拠**: F7 最良 = tilt=0.50/cap=0.10 → Sharpe=+0.905 で PASS閾値
+0.9115 に対し残 0.006。cap 拡張で残ギャップを埋められるか検証する。

**サニティ**: REF CAGR_OOS=+33.53% (diff vs E4 best CAGR=+0.00pp)

## §4 考察

F7 (cap=0.10) からの拡張で、cap が制約として効いていたかを確認する設計:
- cap が大きいほど高確信 bull 日のアップサイドを取りに行ける反面、
  bond クッションの食い込みが深くなり MaxDD は悪化方向
- ピーク tilt_amount = tilt × 0.25 × 0.5 = 0.125 × tilt（raw_a2=0.575 付近）
  → tilt=0.5 ではピーク 0.0625 で cap=0.10 にも届かない。ただし raw_a2 が
    THRESHOLD 寄り or 1 寄りでない中央付近の "厚い bull" 帯では cap が支配的
- 自由度 2（tilt, cap）。F7 比でわずかに過学習リスクは上がるが、cap=0.10 の
  F7 結果と一貫していれば信頼性は担保される
- Trades/yr は wn の連続変化により REF より増加する点に注意

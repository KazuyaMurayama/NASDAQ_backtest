## 3. 6次元スコアカード — 配分と採点

### 3.1 重み配分（あなたの選好を反映：CAGR優先・5y/10y重視・DD許容だが無制限でない・コスト意識・過学習回避）

> 設計思想を3グループに整理。**GET(得るもの)を重め、TRUST(信頼性)とCOSTは guardrail として確実に効かせる**。

| グループ | 次元 | 指標 | 重み |
|---|---|---|---:|
| **GET（リターン）** | ① リターン水準 | min(IS,OOS) と WFA CI95下限 | **22%** |
| | ④ テール（5y/10y） | Worst10Y★・P10_5Y・Worst5Y | **20%** |
| **TRUST（信頼性/過学習防止）** | ② 頑健性 | CPCV p10・Regime_min・WFE | **20%** |
| | ⑥ 有意性 | bootstrap対base・permutation | **12%** |
| **COST（コスト/リスク）** | ③ リスク | MaxDD・Sharpe | **14%** |
| | ⑤ コスト | Trades/yr・実効>3×比率 | **12%** |
| | | **合計** | **100%** |

> GET=42% / TRUST=32% / COST=26%。リターン寄りだが、リスク・コストは0でなく**guardrail**として機能。

### 3.2 ハード・ベト（配点とは別の失格条件）— 「高DD・過学習の怪物」を点数で通さない

- MaxDD が **−50%より悪い** → 失格
- WFE > 1.5（regime luck）→ 失格
- Worst10Y★⓽ < 0 → 失格
- Regime_min が **−10%より悪い** → 失格

→ **vz065_l5 は MaxDD −59.08%（ベト）かつ Regime_min −14.89%（ベト）で失格**。点数に関係なく主候補から除外。

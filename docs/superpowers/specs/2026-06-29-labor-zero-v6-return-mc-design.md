# 労働補填ゼロ v6【リターン経路の確率化】設計 — labor=0 達成「確率」を100%へ近づける

作成日: 2026-06-29
最終更新日: 2026-06-29

## 0. 依頼（ユーザー 2026-06-29）

1975-2020 を開始年とし、次の条件でシミュレーション:
1. 支出: 毎年 **720万円（名目固定）**
2. インフレ: 年率 **2%** を前提
3. 目標: **いかに労働をゼロにできるか、その確率を100%に近づけるための方法**を検討

資産4,000万を「運用用」と「補填用（債券運用等）」に分割する方式を、過去検討と同様に継続。
あわせて現プランの**問題点・脆弱性・改善余地を幅広く多数**挙げ、改善プランを提示。
プラン構築後はユーザー確認を待たず、Claude推奨に沿って実行してよい。

## 1. 条件の再解釈（第一原理）

過去検討で確定した2事実から、条件を厳密化する。

- **(A) 「確率」を測るにはリターン経路の確率化が必須**。史実リターン1本のままでは labor=0 は「46開始年中いくつ」という決定論的カウントにしかならない（v3/v4/v5 はこれ、または**インフレのみ**確率化）。「労働ゼロの確率」を出すには**リターン経路を確率化**する。
- **(B) 「名目固定720万＋インフレ2%」は実質維持(v4)と別物**。今回ユーザーは**名目固定**を明示。名目固定なら実質購買力はインフレで痩せるが、**取り崩し負担は year0 の18%から年々低下**（資産は複利成長・支出は名目一定）。これは labor=0 に**有利**。インフレ2%は本検証では「リターンの実質評価」と「終端購買力の実質表示」にのみ使い、**支出の名目額は720万固定**。

→ **v6 = 名目固定720万・インフレ2%（実質評価/表示用）・リターン経路をブロックブートストラップで確率化し、P(labor=0) を測り、運用/補填分割の各レバーでそれを最大化する。**

## 2. アプローチ選定

| | 案 | 採否 | 理由 |
|---|---|:--:|---|
| ① | リターン・ブロックブートストラップMC | **採用** | 「確率を100%へ」に唯一直接答えられる。ブロック法はシーケンスリスク/暴落クラスタを保存＝経路依存サバイバル指標に正しい（R-STAT-1/2 整合） |
| ② | 史実1本・開始年スイープ | 不採用 | 確率でない（v3/v4 で既出） |
| ③ | パラメトリック正規MC | 不採用 | 厚い尾を捨てテールを過小評価 |

## 3. 確率の評価単位（設計判断・確定）

- **主指標 ❷**: 試行単位 = (開始年, ブートストラップ経路)。「ある開始年に引退したら労働ゼロになる確率」= 個人が直面する確率。**P(labor=0) = labor0 となった試行 / 全試行**。
- **補助 ❶**: 経路単位「経路内で全46開始年 labor0 か」（過度に厳しい AND・参考のみ）。

主指標を ❷ とするのは、ユーザーの問い（労働をゼロにできる確率）に最も忠実なため。確認を取らず採用。

## 4. シミュレーション仕様

- **運用スリーブ**: P09_STR sc2.6（v3/v4 と同一）。年次後税リターン。
- **補填スリーブ**: 1倍債券（基準）。混合（金/現金）はレバーとして探索。
- **支出**: 毎年 720万（名目固定・strict spend）。total < spend なら labor++（その年は total を使い切る）。
- **分割/投入ルール**: v3 ハーネスの tranches（しきい値降順・各 tranche は then-remaining の frac を投入）。基準は v2 幾何 [(14M,1.0)]。hold-if-crash（運用がマイナス年は投入見送り）をレバー化。
- **リターン確率化（ブロックブートストラップ）**:
  - 運用と補填の**同年リターンを同時にサンプリング**（年単位でペアを保つ＝株債の同時性を保存）。
  - **ブロック長 L 年**（基準 L=5、感度 L∈{3,5,8}）の連続塊を史実 1975-2025 から復元抽出し、必要年数まで連結。
  - 各経路で 46 開始年相当を評価（開始年= 経路の各位置）。
  - N 経路（基準 N=1000）。固定シード。
- **インフレ**: g=2% 定数。終端購買力の実質表示と「実質取り崩し負担」注記に使用。支出名目額には**かけない**（名目固定が条件）。

## 5. 探索レバー（＝問題点・脆弱性への対処）

| # | 脆弱性（過去検討で判明） | レバー |
|---|---|---|
| 1 | シーケンスリスク（初期暴落で予備枯渇・2012-14型） | run-fraction、分割投入 vs 全額投入 |
| 2 | 株債同時安（2015型・予備の生命線が効かない） | 予備混合（債券/金/現金） |
| 3 | 暴落年の予備浪費 | hold-if-crash |
| 4 | 固定額の硬直性（P(labor0) の天井） | **保証フロア併用版**も測り「名目固定720万を貫く上限 P」vs「フロア入りで100%にどれだけ近づくか」を定量比較 |
| 5 | インフレで実質が痩せる | 終端**実質購買力**を併記しトレードオフ明示 |

## 6. 検証・QC

- **セルフテスト**: ブロック長=全期間（＝史実1本に縮退）で v3/v4 の labor 数・floor を再現（土台検証）。再現しなければ HALT。
- **独立QC**: ヘッドライン P(labor=0) を別実装（sim を import せず再計算）で突き合わせ。
- **方法論整合**: R-STAT-1/2/3（経路依存指標はブロック法・無傷の危機窓・時機 vs デレバ）を遵守。block≪危機長の単年シャッフルは使わない。
- **生計測値のみ**: 合否ラベルを付けず、P(labor0)・破綻率・実質終端・カット年数の生値で提示。

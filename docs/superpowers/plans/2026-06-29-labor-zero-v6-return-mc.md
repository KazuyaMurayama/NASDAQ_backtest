## ゴール
名目固定720万・インフレ2%（実質評価用）で、リターン経路をブロックブートストラップ確率化し
**P(労働=0)** を測定→100%へ近づける運用/補填分割と支出ルールを探索。

## ステップ（実施済み）
1. ✅ v3ハーネスを import するv6本体 `labor_zero_v6_return_mc_20260629.py` を実装
   - `get_paired_history`: 運用sc2.6・補填債券の同年リターンをペアで取得
   - `make_block_path`: ブロック復元抽出（同年ペア保存）
   - `sim_one`: 1引退の strict-spend sim（tranches・hold-if-crash・floor/top_wr 対応）
   - `mc_prob`: N経路MCで P(labor0) 等を集計
   - セルフテスト: block=全史実・原順で v3 を再現（1975 labor0 / 2012 labor8）
2. ✅ ROUND 0-3 を実行
   - R0: ブロック長感応（3/5/8）
   - R1: 運用/補填比 × hold-if-crash
   - R2: 分割投入 × 予備混合（金/現金）
   - R3: 保証フロア（資産連動）で P→100%
3. ✅ 独立QC `labor_zero_v6_qc_20260629.py`（sim+bootstrap を再実装しヘッドライン再現）
4. ✅ シード安定性チェック（5シード）
5. ✅ レポート `LABOR_ZERO_V6_RETURN_MC_20260629.md`
6. ✅ commit → URL検証 → 3列表報告 → メモリ更新

## 主要結果
- 名目固定720万: P(労働=0) ≈ 0.88（0.875–0.898）が天井。100%不能。
- フロア360万＋好機720万: P(労働=0)=99.85–100%。
- 分割投入は逆効果・予備混合は中立・運用比は鈍感（過去結論を確率版で再確認）。

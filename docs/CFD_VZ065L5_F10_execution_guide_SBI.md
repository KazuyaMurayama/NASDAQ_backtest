## 3. シグナルと目標レバ（通知ベース）

本戦略のシグナルは **別途構築するGAS** が日次で自動算出し、結果を **LINE / Gmail** に通知する（前回ベスト戦略 `NASDAQ-strategy-gas` と同方式）。利用者は通知を受け取り、§B に従って建玉を調整する。

```
[GAS 日次トリガー]
  DataFetch : NASDAQ終値 + VIX（Yahoo Finance）
  Signals   : vz(thr 0.65) + LT2-N750 + F10(ε 0.015) → 目標レバ L_t（1.0〜5.0, DELAY=2適用後）
  Notify    : LINE + Gmail に「目標レバ・必要証拠金・建玉アクション」送信
[利用者]
  通知受信 → §B で 米国NQ100 CFD の建玉を L_t に調整
```

- **L_t の範囲**: 1.0〜5.0倍（防御局面ほど低レバ、トレンド良好＋低VIXほど高レバ）。
- GAS本体は本ガイドのスコープ外（後日 `NASDAQ-strategy-gas` のブランチ/サブフォルダで実装予定）。それまでは同リポのロジックを手計算で代替してもよい。

## A. 初回セットアップ（1回だけ）

### A-1. SBI証券CFD口座を開設
- 通常の証券口座とは別に **CFD取引口座** の申込・審査が必要。

### A-2. CFD取引ルールの同意・リスク確認
- レバレッジ・ロスカット・建玉金利の説明事項に同意。

### A-3. 証拠金（円）を入金
- CFD口座へ円で入金。**為替両替は不要**。
- 目安：想定最大建玉（総資産 × 5.0）の **10% + 余裕バッファ**（§E参照）。

---

## B. 毎回の運用フロー

### B-1. 目標レバ L_t を確認
- GAS通知（LINE/Gmail）から当日の **L_t** を確認。

### B-2. 目標建玉・必要枚数を算出
```
目標建玉（円） = 総資産（証拠金評価額） × L_t
必要枚数        = 目標建玉 ÷ 約420万円/枚   （切り捨て）
必要証拠金      = 目標建玉 × 10%
```

### B-3. 現在の建玉枚数を確認
- SBI CFD画面で現ポジション（枚数）を確認。

### B-4. 差分のみ発注
- `必要枚数 − 現枚数` を売買。増レバ→買建、減レバ→決済（売）。
- **減らす（決済）を先に**実行して証拠金を確保してから増やす。

### B-5. 約定確認
- 約定枚数・平均建値・必要証拠金・**証拠金維持率** を確認。

### B-6. 維持率チェック（§E）
- 発注後の維持率が安全圏（推奨 **200%以上**）にあるか確認。

---

## C. リバランス時の注意

1. **発注トリガー**: L_t が前回適用値から **±0.25倍以上** 変化した場合のみ建玉を調整（過剰売買・スプレッド負けを防止）。
2. **DELAY=2**: 通知の L_t は既に2営業日シフト適用済み。利用者側で追加シフトしない。
3. 減レバ（決済）→ 増レバ（建増し）の順で実行。
4. **米国休場日**: NQ100 CFDは原則取引不可/流動性低下。発注は通常取引時間に。
5. **税務**: 申告分離課税。年内の決済損益を損益通算・繰越（最大3年）可能。確定申告で対応。

## E. 証拠金・ロスカット管理 ★重要（新設）

CFDはETFと異なり **強制ロスカット** がある。本戦略は最大5.0倍だが、商品上限10倍に対する余裕を常に確保する。

| 項目 | SBI 米国NQ100 CFD |
|------|-------------------|
| 必要証拠金率 | 10%（＝建玉の10%） |
| ロスカット発動 | **証拠金維持率 100% 割れ** |
| 追証猶予 | **なし（自動決済）** |

### 維持率の考え方
```
証拠金維持率 = 証拠金評価額（equity） ÷ 必要証拠金 × 100%
必要証拠金   = 建玉時価 × 10%
```

### 安全運用の指針
- 戦略上限 **L_t=5.0** でも、商品上限10倍の **半分** に留まる設計（建玉/equity=5 → 維持率 約200%スタート）。
- L_t=5.0 時、NASDAQの**当日約12〜13%の逆行**で維持率100%接近の目安。低レバ局面ほど耐性は大きい。
- **余裕バッファ**: 証拠金は「必要証拠金（最大建玉時）＋追加20〜30%」を常時確保。
- 急落時はGAS通知が低レバ L_t を出すまでのラグ（DELAY=2）に注意。維持率が150%を割ったら手動で一部決済を検討。

> ⚠️ 維持率・ロスカットの厳密な計算と最新の証拠金率はSBI取引画面の表示を**必ず一次確認**すること（上表は概算前提）。

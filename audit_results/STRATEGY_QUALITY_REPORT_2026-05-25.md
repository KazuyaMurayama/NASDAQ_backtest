## 1. 戦略の分解（6 レイヤー構造）

E4 Regime k_lt 戦略は **6 つのレイヤーを直列合成**した構造を持つ。各レイヤーが独立したアルファ仮説に対応する。

### Layer 1: ベースシグナル — DH Dyn A2 Optimized

```
構成要素:
  DD (Drawdown)           ─┐
  AsymEWMA (非対称EWMA)    ─┤
  TrendTV (トレンド/TV)    ─┤──→ 重み付き合成 → threshold=0.15
  SlopeMult                ─┤                    ↓
  MomDecel (モメ減速)      ─┤              lev_A ∈ {0, 0.25, 0.5, 0.75, 1.0}
  VIX_MR (VIX平均回帰)     ─┘              （27回/年リバランス）
```

- **役割:** 短〜中期のリスクオン/オフ判定
- **アルファ仮説:** マルチファクタの非線形合成で局面転換を捉える
- **試行数寄与:** 6 ファクタ × 重み最適化 → 主要な選択バイアス源（推定 N=300〜400）

### Layer 2: LT2 長期モメンタムシグナル

```
lt_sig_t = compute_lt2(close, N=750)
         = z-score(close_t / SMA(close, 750) − 1)
```

- **役割:** 750 日（約 3 年）モメンタムによる長期トレンド方向判定
- **アルファ仮説:** 長期モメンタム異常（Jegadeesh & Titman 1993 系列）
- **N=750 の意義:** ロングルックバックでパラメータ感度を抑制（短期: N=63, 中期: N=252 と比較し最も安定）

### Layer 3: E4 Regime-Conditional k_lt（★新機構）

```
vz_t = rolling_zscore(realized_vol_20d, window=252)

k_lt_t = ┌ k_lo = 0.1   if vz_t < −vz_thr (−0.7)  [低ボラ局面: 相場安定]
         ├ k_mid = 0.5  if |vz_t| ≤ vz_thr        [中間レジーム]
         └ k_hi = 0.8   if vz_t > +vz_thr (+0.7)  [高ボラ局面: 相場不安定]

lt_bias_t = clip(−k_lt_t × lt_sig_t × 0.5, −0.5, +0.5)
lev_mod_t = clip(lev_A_t + lt_bias_t, 0, 1)
```

- **役割:** ボラレジームに応じて LT2 の影響度を動的調整
- **旧戦略との差分:** 旧 = 固定 k=0.5（全レジーム同じ）→ 新 = レジーム条件付き
- **アルファ仮説:** 低ボラ時は LT2 の反転寄与を弱め、高ボラ時は強化することでドローダウン改善
- **新リスク:** vz レジーム判定の自己参照性（→ Q2-Type4 で詳述）

### Layer 4: CFD S2 動的レバレッジ（VZ-Gated）

```
L_s2_t = compute_L_s2_vz_gated(
    ret, vz,
    target_vol=0.8, k_vz=0.3,
    gate_min=0.5, n=20,
    l_min=1.0, l_max=7.0, step=0.5
)
```

- **役割:** 実現ボラティリティに基づく 1x〜7x の動的レバレッジ調整
- **アルファ仮説:** ボラ目標化（Moreira & Muir 2017）
- **VZ ゲート:** ゲート未充足時はレバレッジ縮小（守りのバイアス）

### Layer 5: ポートフォリオ配分（DH Dyn [A]）

```
(wn, wg, wb) = simulate_rebalance_A(...)
  wn: NASDAQ 比率
  wg: Gold 2x 比率   (TER 0.50% sim proxy)
  wb: Bond 3x (TMF) 比率 (TER 0.91%)

r_portfolio = wn × (lev_mod × L_s2 × r_nas_cfd) + wg × r_gold2x + wb × r_tmf
```

- **役割:** 三資産分散でリスク平準化
- **注意:** Gold 2x の sim TER (0.50%) と実 TER (0.95%) の差 = **−10.5bps/yr** はコスト未計上リスクとして CAGR にオフセット要

### Layer 6: DELAY = 2 営業日

```
position_t = signal_{t−2}
```

- **役割:** look-ahead bias 防止
- **意義:** IS-OOS gap がマイナスでも「データリークではない」根拠

## 2. Q1: パフォーマンスリスクの定量化

### 2.1 IS-OOS gap = −1.81pp の 3 通りの解釈（★新発見）

旧戦略では IS-OOS gap = +0.18pp（IS と OOS がほぼ一致 = 健全）だったが、E4 では **−1.81pp（OOS > IS）** という逆転現象が起きた。これは通常の「過学習なら OOS < IS」のセオリーに反する重要な観察。

#### 解釈 ①：良い解釈 — 真の汎化（推奨）

- **論拠:** WFA Post-IS 期間の平均 CAGR = +46.70%（IS 平均 +41.30% より +5.40pp 高い）が独立に支持
- **メカニズム:** E4 のレジーム条件付けが「未知のボラ環境」に適応し、ロバスト性を高めている
- **整合性:** DELAY=2 でデータリークは排除済み

#### 解釈 ②：中立解釈 — OOS 期間のレジーム的有利

- **論拠:** OOS 期間（2021–2026）は NASDAQ にとって特殊：
  - 2021: コロナ後の流動性相場
  - 2022: 金利急騰局面（戦略はこれをうまく回避）
  - 2023–2024: AI バブル
  - 2025: 高金利持続
- **メカニズム:** E4 の k_hi=0.8（高ボラ強反転）が 2022 の急落を捉えた可能性
- **リスク:** 将来の異なるレジームでは同等の優位は再現しない可能性

#### 解釈 ③：悪い解釈 — IS 終端領域のデータリーク

- **論拠:** vz の rolling zscore (window=252) が IS/OOS 境界で「未来情報」を参照する可能性
- **反論:** DELAY=2 と rolling 計算は過去 252 日のみ参照 → 構造的にリーク不可能
- **結論:** リスクは低いが、付録 9.3 の「Layer 3 自己参照性検証」で再確認推奨

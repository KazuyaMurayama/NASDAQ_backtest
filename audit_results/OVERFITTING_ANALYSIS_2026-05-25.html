Q2: どんなタイプの過学習か？

過学習を3タイプに分解して診断。

Type 1: 選択バイアス / 多重比較バイアス（主因）

DSR (Deflated Sharpe Ratio) による検定:

試行数 NE[max SR]DSR判定
N=50 (楽観)≈ 1.840.00106FAIL
N=150 (中央)≈ 2.494.3e-05FAIL
N=500 (保守)≈ 2.881.0e-06FAIL
PSR (SR_b=0)—0.969PASS
PSR (SR_b=0.5)—0.781PASS

読み方

  • 単試行・帰無SR=0 なら PSR=0.969 で十分有意。
  • しかしプロジェクト全体で ~500 試行から最良を選んだ場合、運だけで E[max SR] ≈ 2.88 まで到達しうる。
  • 観測 0.858 はその閾値を大きく下回り、「500試行最良の中では平凡」な水準。
  • 統計量: skew(γ3)=-0.516, kurtosis(γ4)=8.99 → 左裾厚 + 高い尖度で、PSR/DSR が下方に補正される構造。
結論: これが最大の過学習リスク。試行回数削減と仮説事前登録が必要。

Type 2: パラメータ過学習（古典的）— 軽微

  • IS Sharpe: +0.97 相当（10分割平均）
  • OOS Sharpe: +0.858
  • CAGR_IS = +31.3% vs CAGR_OOS = +31.2%
  • IS-OOS gap = +0.18pp（極めて小さい）
判定: 古典的な「IS で過剰最適化 → OOS で崩壊」型ではない。LT2-N750 のロングルックバックがパラメータ感度を低くしている。

Type 3: レジーム依存性 — WARN

Permutation Test の解釈で最も重要な発見:

検定対象p値置換 mean Sharpe解釈
(a) L_s2 ブロックシャッフル動的レバ調整 0.2480.763 L_s2 を壊しても 0.763 残る → L_s2 はほぼ無価値
(b) L_s2 White-noise 復元同上（参考） 0.3100.768 同様
(c) lev_mod ブロックシャッフル参加タイミング 0.0550.547 lev_mod を壊すと BH(0.516) レベルに低下 → これが主アルファ源
重要な発見
L_s2 を完全にランダム化しても置換分布の平均 Sharpe は 0.763 で、BH 1x (+0.516) を大幅に上回る。これは戦略のアルファのほぼ全てが lev_mod（市場参加タイミング）に由来し、L_s2（動的レバレッジ調整）はほとんど寄与していないことを示す。
これは「過学習」というより「アルファ源の誤認」。lev_mod が2021-2026の特定レジームに有利だったため Sharpe が嵩上げされている可能性があり、レジームが変わると lev_mod (c) の p=0.055 の境界値が失効する恐れ。

Q3: 対策はあるか？

3.1 期待値の修正（最優先）

  • 公式 Sharpe 期待値: 観測 +0.858 ではなく Bootstrap CI95 下限の +0.086〜+0.131 を「悲観ケース」として併記。
  • 公式 CAGR 期待値: SBI-CFD 高推定 (+25.4%) を「保守ケース」とし、運用判断はこの数値で行う。
  • DSR FAIL を踏まえ、N=150〜500 の選択バイアス補正後の「真の期待 Sharpe」はおおむね 0.3 前後と想定するのが安全。

3.2 ポジションサイジング縮小

  • 観測 Sharpe 0.858 を信じた Kelly 比率ではなく、CI95 下限基準でハーフ Kelly or クォーター Kelly に落とす。
  • 推奨: 想定レバレッジを現行の半分（例: 2.0x → 1.0x）で実運用検証。

3.3 レジームモニタリング

  • lev_mod のシグナル発生頻度を月次でトラッキング。
  • 過去3ヶ月の lev_mod 平均 vs 全期間平均が ±30% 乖離したら警告。
  • VIX レジーム / VZ レジームに応じた lev_mod の動作監査。

Q4: どのように改善すべきか？

4.1 lev_mod に集中する

Permutation (a) と (c) の結果は明確:

構成要素寄与度（推定）対応
lev_mod（参加タイミング）アルファのほぼ全て最優先で改善
L_s2（動的レバ調整）ほぼゼロ簡素化または除外を検討

具体アクション:

  • L_s2 を固定レバレッジ（k=0.5 定数）に置き換えて再評価し、Sharpe 低下が小さければ正式に簡素化。
  • lev_mod の False Positive / False Negative 分析（どの相場で誤判定したか）を実施。

4.2 OOS 延長モニタリング

  • 四半期ごとに Sharpe_OOS_rolling_4Q をトラッキング。
  • 直近4四半期の Sharpe が 0.3 を下回ったら戦略停止 or レビュー。
  • 累積 OOS 期間を5年・7年・10年と延伸し、DSR の実質的な E[max SR] を引き下げる。

4.3 モデルアンサンブル

  • 単一パラメータ (N=750, k=0.5) ではなく N∈{500, 750, 1000}, k∈{0.3, 0.5, 0.7} の組合せ平均でポジション決定。
  • パラメータ感度を低下させ Type 1 (選択バイアス) を緩和。

Q5: 汎化性能を高めるには？

5.1 試行数削減（最重要）

  • 次回の戦略設計フェーズでは仮説数を N ≤ 50 に絞り、DSR の E[max SR] を 1.84 程度に抑制。
  • 仮説事前登録（Pre-Registration）: 各仮説の論理的根拠を文書化してから検定実行。
  • グリッドサーチを「網羅探索」から「仮説駆動探索」へ転換。

5.2 事前確認: 同時置換検定 (d) の追加実施

  • 現在の (a)(b)(c) に加え、L_s2 と lev_mod を同時にブロックシャッフルする検定 (d) を追加。
  • 同時置換後の Sharpe が BH 1x (+0.516) を下回るかで「真の戦略全体アルファ」を測定。
  • 検定 (d) で p<0.05 を満たせば、本戦略の信頼性は再評価可能。

5.3 コスト頑健性を主評価に

  • 現行モデル (0.20%/yr) は理論上限。実運用では SBI SOFR+1〜3% を主シナリオに。
  • EVALUATION_STANDARD.md の9指標標準に「ブローカー実効コストでの再計算」を追加し、PASS 条件を厳格化。
  • Worst10Y★ の判定閾値をブローカー別に表示し、運用商品選択を意思決定の前段に組み込む。

## §3 振り返り分析（5 つの問い）

### Q1: なぜ B4 (k_lo=0.0) は効果がなかったのか？

**構造的理由**: E4 の弱気時 (vz < -vz_thr=-0.7) は **既に LT2 バイアスが 5% (k_lo=0.10 × 0.5 = 0.05)** とほぼ無視できる強度である。

- `lt_bias = -k × lt_sig × 0.5` のため、k=0.1 なら最大 ±0.05 のレバ補正 = レバ 5.0±0.25 程度。
- これを k=0.0 にしても **削減できるスワップは 0.05 ×(年間における低ボラ期割合 ≈ 15%) ≒ 0.75bps/yr** で誤差レベル。
- CSV 確認: k_lo=0.0 vs k_lo=0.1 の Sharpe 差 = -0.005、CAGR 差 = -0.23pp、MaxDD 差 = +0.4pp（むしろ悪化）。
- **示唆**: 「完全クローズ」は 弱気時 LT2 が 既にほぼ無効化されているため、改善余地が物理的に存在しない。**弱気時の改善は k ではなく l_max (レバ上限) を操作する方向 (= A2) が正解の方向性**。

### Q2: なぜ A1 (sigmoid) は MaxDD が悪化したのか？

**機構**: sigmoid 連続化は **vz∈(-0.7, +0.7) の "中間域" を撤廃** し、vz=0 付近では k=0.45 (sigmoid 中央値) を返す。E4 離散版は中間域で固定 k=0.50 を維持 する。

- α=2 で k(vz=0)=0.45、α=8 で k(vz=0)=0.45（sigmoid 中心値は α 不変）。
- しかし α が大きいほど **vz が ±0.5σ 付近で急峻に k_hi/k_lo へ遷移** し、高ボラ期 (vz>+0.5σ) で k_hi=0.8 が長期間維持される。
- k_hi=0.8 は **下落トレンド時に LT2 バイアスを -0.4 まで強化** = レバ実効値 ≈ 5.0 × (1-0.4) = 3.0 まで下げる。これは「ボラ天井で機械的に売る」動作で、**反発局面 (例 2020-03〜04) でレバ回復が遅延 → drawdown 拡大**。
- CSV: α=100→69.9%, α=8→67.1%, α=5→65.0%, α=3→62.9%, α=2→61.6%。**MaxDD は α と単調に悪化**。
- **示唆**: E4 離散版の vz_thr=0.7 という保守的な閾値は **境界付近のホイップソー回避が目的ではなく、レジーム切替の頻度抑制が目的**であった。sigmoid は逆に切替頻度を上げてしまう。

### Q3: なぜ A2 (Dynamic l_max) は Sharpe が改善しなかったのか？

**根本原因**: VOL_REF=0.20 (年率20%) という基準値が **NASDAQ 1974-2025 の vol252 長期平均 0.183 より高い** (差 +1.7pp)。

- `l_max_t = clip(lmax_base - vol_sens × (vol252_t / 0.2 - 1), 4.5, 6.5)`
- 平常時 vol252=0.183 のとき、vol_sens=2.0 なら補正項 = 2.0 × (-0.085) = -0.17 → l_max が **平常時に lmax_base から 0.17 下振れ**。
- これは「凪の時に余裕を持つ」が逆向きに作用し、**低ボラ期に CAGR を毀損**。CSV 確認: 9/9 config で MaxDD 改善するが CAGR は -1.0〜-4.7pp 低下。
- ベスト config (base=6.0/sens=2.0) は lmax_base を 7.0 → 6.0 にしただけで、実質「l_max 単純引き下げ」と区別困難。
- **示唆**: VOL_REF を **動的中央値 (vol252 の rolling median = 0.183)** に置き換えれば対称な動作になる。次の実験候補。

### Q4: なぜ C3 (Yang-Zhang) は FAIL したのか？

**数学的理由**: Yang-Zhang ボラ推定量は close-to-close より分散が約 1/8 と低いが、本実装では **Z-score 化 (current_yz / 252日 std_of_yz)** で標準化している。

- Z-score 化により **絶対水準の情報が消失**し、相対変動だけが残る。
- YZ の精度向上の本質は「平常時の分散が小さい → スパイクが SN 比的に検出しやすい」だが、これは **絶対値ベースの閾値**でないと活かせない。
- 同じ vz_thr=0.7 は close-to-close では「σ_20d の +0.7σ 超」を意味するが、YZ では「σ_yz_20d の +0.7σ 超」を意味し、**統計的には異なるイベント**になる。CSV: YZ_n=10/vz=0.7 で Sharpe=0.888（E4=0.891 を僅か 0.003 下回る）。
- 1974-1984 の合成 OHLC (20.7%) が YZ の openinng-gap 項を **常に 0** にする → 推定がバイアスを持つ可能性も。
- **示唆**: 「ボラ推定量を変える」のではなく「Z-score の代わりに **vol level (絶対水準)** をフィーチャに使う」のが正しい方向。

### Q5: なぜ C2 (Adaptive Deadband) は固定 ε と同等だったのか？

**統計的理由**: `vol_ratio = σ_20d / σ̄_250d` の長期平均 = **1.010 ≈ 1.0**。

- 定義上 σ_20d は σ̄_250d の構成要素の一部 → 母集団的に E[σ_20d/σ̄_250d] ≈ 1.0。
- vol_ratio が clip 範囲 [0.3, 3.0] で動いても、**ε_t = ε_0 × vol_ratio の時系列平均 ≈ ε_0** で、deadband の有効幅は固定 ε とほぼ同じ。
- 結果: Trades/yr=51.3 vs F10 51.6 (-0.3 回)、Sharpe 同等 (差 +0.000)。
- **示唆**: adaptive にするなら **平均が 1 でない統計量** (e.g. ATR/price、または volatility-of-volatility) を使う必要がある。CFAR の意義を活かすには「ノイズ分布の絶対水準」を反映する量が必要。

## §4 次の実験推奨 Top 3

### #1 (推奨): **動的 VOL_REF (rolling median) 版 Dyn l_max**

**根拠**: Q3 で判明した「VOL_REF=0.20 固定 → 平常時に必ず l_max が下振れ」問題を解消。
- 実装: `vol_ref_t = rolling_median(vol252, window=2520)` (≒ 10年) に置換
- 推定 CAGR_OOS: +33.0〜34.0%, Sharpe_OOS: +0.90〜+0.92, MaxDD: -57〜-58%
- 実装難易度: **低** (a2_dyn_lmax.py の VOL_REF を 1 行差し替え)
- 期待: A2 が捕まえた MaxDD 改善 (4.7pp) を CAGR を犠牲にせず実現できる可能性。

### #2: **E4 + vol level deadband (ε_t = const × σ_20d_level)**

**根拠**: Q5 で判明した「vol_ratio の平均が 1.0 で adaptive 化が無意味」問題を解消。比率ではなく **σ_20d の絶対水準** を deadband に使う。
- 実装: `ε_t = c × σ_20d` (c は 0.06 / 0.08 / 0.10 などで sweep)
- 推定 CAGR_OOS: +35〜+37%, Sharpe_OOS: +0.94〜+0.96, MaxDD: -60〜-63%
- 実装難易度: **低** (c2_adaptive_deadband.py の ε_t 計算を vol_ratio → σ_20d 直接参照に変更)
- 期待: 高ボラ期 (σ_20d=0.40) では ε=0.024 → 過剰取引抑制、低ボラ期 (σ_20d=0.10) で ε=0.006 → ピックアップ感度向上。F10 を超える Sharpe が出る可能性。

### #3: **E4 + 非対称 k_lt (k_hi だけ動かす)**

**根拠**: Q1 で判明した「弱気時 k は既にほぼ無効、改善余地は高ボラ側」「Q2 で k_hi=0.8 が反発局面の drawdown を悪化させる」の 2 つの知見を組み合わせ。
- 実装: k_lo=0.10 固定、k_hi ∈ {0.5, 0.6, 0.7} sweep。高ボラ時 LT2 バイアス強度を **緩和** する方向。
- 推定 CAGR_OOS: +33〜+34%, Sharpe_OOS: +0.90〜+0.92, MaxDD: **-56〜-58%** (反発捕捉が改善)
- 実装難易度: **低** (b4_klo_zero_sweep.py の grid を k_lo=[0.1] / k_hi=[0.5,0.6,0.7] に変更)
- 期待: MaxDD 改善（A2 と同方向）と CAGR 維持（A2 の弱点克服）の両立。

## §5 教訓まとめ (5 つの Anti-Pattern)

| # | Anti-Pattern | 教訓 |
|:-:|:-------------|:-----|
| 1 | 既にほぼ無効化されているパラメータを更に絞っても改善しない (B4) | 介入の "限界効用" を事前に計算 |
| 2 | 離散→連続化が常に改善とは限らない。離散化が "頻度抑制" を目的にしている場合は逆効果 (A1) | 既存パラメータの設計意図を理解してから連続化を試す |
| 3 | 固定基準値 (VOL_REF, σ̄) の平均が母集団平均とズレていると adaptive 化が一方向に偏る (A2) | 基準値はデータの長期統計から動的に取る |
| 4 | Z-score 化は絶対水準情報を捨てる → ボラ推定量の精度向上が無効化される (C3) | 標準化前のリレベル情報が重要なら、Z-score 化しない |
| 5 | 比率ベース adaptive は分子分母が同源の場合 ratio≈1.0 で無意味になる (C2) | adaptive 化する量は分母から独立な統計量を使う |

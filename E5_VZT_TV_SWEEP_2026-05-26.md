## §4 重要発見

### §4.1 vz_thr=0.70 の頑健性 — **再評価必要**

- 連続スイープ 9 点中の Sharpe ランキング:
  1. **vz_thr=0.65 → Sharpe=+0.945** (CAGR=+36.99%, MaxDD=-58.68%, W10Y★=+18.74%, **IS-OOS gap=-5.22pp ⚠**)
  2. vz_thr=0.70 → Sharpe=+0.891 (baseline, IS-OOS gap=-1.81pp)
  3. vz_thr=0.60 → Sharpe=+0.886
- baseline (vz_thr=0.70) と最良値 (vz_thr=0.65) の **Sharpe 差は +0.0538**（task 基準 +0.01 の **5.4 倍**）
- これは task §評価基準 (目的1) で「他の値が +0.01 Sharpe 以上優れる → 再評価必要」に明確に該当
- 「離散グリッド (0.3, 0.5, 0.7, 1.0) 上の偶然の勝者」リスクは **顕在化**: 隣接の 0.65 が baseline を有意に上回ることが判明
- 0.55〜0.85 帯の Sharpe カーブは smooth だが、**ピークは 0.65 にあり 0.70 は降下側**
- **⚠ 重要懸念**: vz_thr=0.65 は **IS-OOS gap=-5.22pp** で baseline (-1.81pp) や近隣 (vz_thr=0.55: -0.20pp, vz_thr=0.60: -1.55pp, vz_thr=0.70: -1.81pp, vz_thr=0.75: -0.90pp) と比べ突出して悪い。これは **OOS 期間に偶然のテール上振れが寄った可能性**を強く示唆する
- 0.65 のピークは: (a) 真の最適、または (b) OOS テールイベントによる見せかけのピーク — **WFA で必ず検証する必要**
- 結論: **vz_thr=0.65 は WFA で慎重に検証**（IS-OOS gap が悪いため過学習サイン）。WFE が 0.5 未満になれば偶然の勝者として棄却、0.5〜2.0 にあれば真の改善として採用

### §4.2 tv / l_max トレードオフ実測

- ベースライン (tv=0.80, l_max=7.0): CAGR=+33.53%, MaxDD=-60.01%, Sharpe=+0.891
- 最浅 MaxDD config (tv=0.65, l_max=5.0): MaxDD=-53.27%, CAGR=+27.89%, Sharpe=+0.892
  → MaxDD 改善 ▲6.74pp に対し CAGR 犠牲 -5.64pp（コスト効率: 0.84pp/pp = **悪い**）
- 最浅 MaxDD かつ CAGR ロス最小の組合せ (tv=0.80, l_max=5.0): MaxDD=-54.07%, CAGR=+30.58%, Sharpe=+0.896
  → MaxDD 改善 ▲5.94pp に対し CAGR 犠牲 -2.95pp（コスト効率: 0.50pp/pp = **良好**）
- **l_max を 7.0→5.0 に下げる効果**: 全 tv で MaxDD ▲5〜6pp、CAGR 犠牲 -1.4〜-2.95pp
- **tv を 0.80→0.65 に下げる効果**: 全 l_max で CAGR 犠牲 -2.7〜-4.7pp、MaxDD 改善は意外に小さい（l_max=7.0 で ▲1.31pp のみ）
- 知見: **MaxDD レバーとしては l_max の方が tv より圧倒的に強い**（Round 1B 想定と一致）

### §4.3 「MaxDD -55% & CAGR +30%」達成 config の有無

- 達成 configs: **1 / 12** (tv=0.80, l_max=5.0: CAGR=+30.58%, MaxDD=-54.07%, Sharpe=+0.896)
- Sharpe も baseline (+0.891) を上回る → **shortlist 入り十分価値あり**
- 加えて Part A の vz_thr=0.65 と組合せた未測定領域（vz_thr=0.65, tv=0.80, l_max=5.0）は次ラウンドで検証する価値あり

## §5 採用推奨設定と判断

**Part A 採用判断**: **再評価必要 — ただし慎重に**。vz_thr=0.65 への即時切替えは推奨しない。
- vz_thr=0.65 のメリット: CAGR +3.46pp 改善、Sharpe +0.054 改善、MaxDD ほぼ同水準 (-58.68% vs -60.01%)
- **IS-OOS gap 懸念**: vz_thr=0.65 は -5.22pp で近隣値より突出して悪い → OOS テール由来の見せかけのピーク疑い
- ピーク形状: 0.60→0.886, **0.65→0.945**, 0.70→0.891 とスパイク的（隣接が突出して悪くないため最も低い IS-OOS gap = 最も大きな過学習サインではない可能性も）
- **推奨アクション**:
  1. vz_thr=0.65 を **WFA G2 shortlist に追加**して CI95_lo > 0 & 0.5≤WFE≤2.0 を必ず検証
  2. WFE が 0.5 未満（IS過剰最適）なら棄却し vz_thr=0.70 維持
  3. WFE が 0.5〜2.0 なら真の改善として採用検討
  4. 並行して vz_thr ∈ {0.625, 0.675} の 2 点を追加スイープし、0.65 ピークが滑らかな極大かスパイクか確認

**Part B 採用判断**: tv=0.80, l_max=5.0 を **shortlist 入り推奨**。
- MaxDD -55% かつ CAGR +30% を同時達成する唯一の config
- Sharpe も baseline 同等以上 (+0.896 vs +0.891)
- CAGR 犠牲はわずか -2.95pp で MaxDD ▲5.94pp の改善 → 良好なトレードオフ
- WFA 検証後、MaxDD 重視のバリアントとして CURRENT_BEST_STRATEGY.md の「Defensive Variant」候補に提案

**併用検証推奨**: vz_thr=0.65 × tv=0.80 × l_max=5.0 の未測定 config を次ラウンドで評価。
両改善が独立に効くなら CAGR ≈ +33%, MaxDD ≈ -55%, Sharpe ≈ +0.95 級の可能性。

### §5.1 Round 2B (F10) / Round 2A (B10) への共有知見

- **vz_thr=0.65 が新たな最良候補**: F10/B10 が vz_thr を別パラメータと組み合わせる際、0.70 ではなく **0.65 も同等に基準点として扱うべき**。離散グリッドの「偶然の勝者」リスクが Part A で顕在化したため、F10/B10 のスイープでも vz_thr は連続値で扱う必要あり
- **tv は MaxDD レバーとして弱い、l_max は強い**: A2 単体の線形外挿（tv=0.8→0.6 で ▲3.5pp）は E4 framework では再現せず（▲1.3pp のみ）。MaxDD 改善は **l_max 削減経由が圧倒的に効率的**。F10/B10 のパラメータ最適化では l_max ∈ {5.0, 6.0, 7.0} を優先候補とすべき
- **同時最適化の未測定領域**: vz_thr=0.65 × l_max=5.0 など Part A 知見と Part B 知見の交差点は本ラウンドで未測定。F10/B10 で取り扱う場合は明示的にカバーすべき

### §5.2 E4 グリッドの追加拡張判断

- **k_hi=0.9/1.0**: Part A で vz_thr 周辺が smooth であることが確認されたため、k_hi 上端 (0.8) 周辺の追加拡張は **限定的 priority**。WFE 過学習リスク増のほうが大きい
- **k_mid=0.4/0.6**: 中間域の k 変化は regime_mid 期間の貢献が大きいため、独立スイープに値する → **別ラウンド (E6 候補) として shortlist 推奨**
- **vz_thr 連続スイープ (Part A) の追加細分**: 0.625 / 0.675 を追加し vz_thr=0.65 ピークの真の頂点を確認することは **WFA 前に推奨**（コストは 2 configs のみ）
